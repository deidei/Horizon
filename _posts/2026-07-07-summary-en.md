## [Tencent Releases Hy3: 295B MoE Model with 21B Active Parameters](https://simonwillison.net/2026/Jul/6/hy3/#atom-everything) ⭐️ 8.0/10

Tencent has released Hy3, a 295B-parameter Mixture-of-Experts model with 21B active parameters and a 256K context length, outperforming similar-size models and available under the Apache 2.0 license. This release demonstrates the growing competitiveness of open-source Chinese AI models and provides a powerful, accessible option for developers and researchers, particularly for long-context tasks. The full model is 598GB on Hugging Face, with an FP8 quantized version at 300GB. It includes a Multi-Token Prediction (MTP) layer with 3.8B parameters, and is available for free on OpenRouter until July 21st.

rss · Simon Willison · Jul 6, 23:57

**Background**: Mixture-of-Experts (MoE) architectures use multiple specialized sub-networks (experts) with a gating mechanism to activate only a subset for each input, keeping computational costs lower than dense models of similar total parameter count. The active parameter count determines inference speed and cost, while total parameters affect storage. Multi-Token Prediction (MTP) layers predict multiple future tokens simultaneously, improving training efficiency and model performance.

## [TRACE: Open-Source Hierarchical Memory Boosts LLM Agent Recall](https://www.reddit.com/r/MachineLearning/comments/1uoz5jo/trace_opensource_hierarchical_memory_for_llm/) ⭐️ 8.0/10

TRACE, a new open-source hierarchical memory system for LLM agents, organizes conversation history into a topic tree with branches and summaries. It achieved 82.5% F1 on the EventQA accurate-retrieval task of MemoryAgentBench using the gpt-oss-20B model. This approach significantly outperforms current flat RAG-based memory systems like Mem0 and MemGPT, which scored 37.5% and 26.2% respectively (with GPT-4o-mini). TRACE demonstrates that hierarchical memory can dramatically improve long-term recall for LLM agents, potentially enabling more capable and persistent AI assistants. The comparison is not fully controlled because TRACE used an open-source model (gpt-oss-20B) while Mem0 and MemGPT used GPT-4o-mini, a different backend. The author attempted to run Mem0 on gpt-oss but faced JSON parsing issues, and omitted Letta due to server setup complexity. Full JSON logs are available in the repository for methodology inspection.

reddit · r/MachineLearning · /u/PsychologicalDot7749 · Jul 6, 14:35

**Background**: LLM agents often struggle with long-term memory, typically using flat retrieval-augmented generation (RAG) from past conversations. Hierarchical memory organizes information into levels of abstraction, such as topics and subtopics, enabling more efficient and accurate retrieval. MemoryAgentBench is a benchmark suite introduced at ICLR 2026 that evaluates various memory capabilities of LLM agents, including the EventQA task which tests accurate retrieval of event details.

## [Claude Cowork sandbox escape vulnerability](https://cyberpress.org/claude-cowork-flaw/) ⭐️ 8.0/10

A sandbox escape vulnerability was discovered in Anthropic's Claude Desktop for Windows, specifically in the Cowork feature, allowing an attacker with local code execution to gain root privileges in the isolated Ubuntu VM. The exploit chain uses DLL sideloading and unvalidated parameters to bypass bubblewrap restrictions and exfiltrate sensitive files. This vulnerability highlights security risks in AI desktop agents that run in sandboxed environments, potentially affecting many users of Claude Cowork. Although the exploit requires prior local access, its technical sophistication and the sensitivity of exfiltrated data (e.g., /etc/shadow) make it a serious concern for enterprise deployments. The vulnerability chain abuses claude.exe's DLL sideloading and two unfiltered parameters (isResume and allowedDomains) in the spawn interface to escape the bubblewrap sandbox and exfiltrate /etc/shadow. Anthropic classified the report as 'not a security issue' because the attacker must already have local code execution.

telegram · zaihuapd · Jul 6, 14:53

**Background**: Claude Cowork is an AI agent by Anthropic that performs office tasks like file editing and desktop organization. It runs in a sandboxed Ubuntu VM using bubblewrap to isolate it from the host system. Bubblewrap is a lightweight sandboxing tool, while nsenter is a utility to run programs in different Linux namespaces, which the exploit uses to break out of the sandbox.
